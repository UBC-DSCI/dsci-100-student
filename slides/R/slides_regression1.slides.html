The blue line depicts our predictions from k-nn regression. The red lines depict the error in our predictions, i.e., the difference between the $i^\text{th}$ test data response and our prediction $\hat{y}_i$:

  • We now have the errors for individual points. How could you combine these errors to report a single measure of error for our model?

1. Is our model any good?¶

We (roughly) add up these errors to evaluate our regression model

  • Not out of 1, but instead in units of the target variable (bit harder to interpret)

Root Mean Squared Prediction Error (RMSPE):

$$RMSPE = \sqrt{\frac{1}{n}\sum\limits_{i=1}^{n}(y_i - \hat{y_i})^2}$$

- $n$ is the number of observations

- $y_i$ is the observed value for the $i^\text{th}$ test observation

- $\hat{y_i}$ is the predicted value for the $i^\text{th}$ test observation
  • Average because we don't want the number of predictions to impact the overall error.
  • Squared diff since error can be positive or egative, could also have used absolute value (different type of metric that is also used)

RMSE vs RMSPE¶

  • Root Mean Squared Error (RMSE): The same formula except on the data used to train the model
    • this indicates how well our model can fit our data
  • Root Mean Squared Prediction Error (RMSPE): predicting on unseen data.
    • this indicates how well our model generalizes to future data

No standard terminology.

Final model from k-nn regression¶

For this model, RMSPE is 91,620.4. How can we interpret this?

  • Prices tend to be +/- 90,000 off that line
  • Not that great for a house that is predicted to cost e.g 200,000 in total
  • What is a good error depends on the application
  • We can't compare 90,000 USD of this model to an error in other units from another model
    • Therefore we sometimes report errors as proportions, but we're no getting into that in this course.

2. How do we choose k?¶

Roughly the same as before!

  1. Cross validation:
    • Split data into $C$ folds
    • Train
    • Evaluate model
    • Pick k that gives the lowest RMSPE on validation set
  1. Train model on whole training dataset (not split into folds)
  1. Evaluate how good the predictions are using the test data
